# A股量化选股系统 (基于 XGBoost)

本项目是一个完整的、端到端的 A 股日频/月频量化选股系统。系统通过拉取真实的历史交易数据，自动计算技术分析因子，利用 XGBoost 机器学习模型挖掘数据规律，并最终在模拟的回测环境中验证策略的有效性。

---

## 1. 项目架构与模块说明

系统主要由 5 个核心 Python 文件组成，它们像流水线一样依次运行：

1. **`config.py`**: 全局配置中心。所有的参数（如时间、资金、因子参数、模型超参数等）都在这里统一管理。
2. **`data_fetcher.py`**: 数据获取模块。自动从 [Tushare](https://tushare.pro/)（主）和 [AKShare](https://akshare.akfamily.xyz/)（备）拉取指定股票池的日线级别行情数据。
3. **`factor_calculator.py`**: 因子计算模块。负责将原始的“开高低收”等行情数据，转换为 XGBoost 模型可以理解的“特征（Features）”和“标签（Labels）”。
4. **`model_trainer.py`**: 模型训练模块。将数据按时间先后划分为训练集和测试集，使用 XGBoost 回归模型（XGBRegressor）进行训练和预测评估。
5. **`backtester.py`**: 回测验证模块。模拟真实的交易环境，根据模型的预测结果进行选股、调仓、扣除手续费，并生成策略资金净值曲线与大盘基准（Benchmark）的对比图。
6. **`main.py`**: 系统的主入口。按顺序调用上述 4 个模块，并支持本地数据缓存（`raw_data.csv`, `factor_data.csv`），大幅提升后续调试速度。

---

## 2. 核心配置文件 (`config.py`) 详解

`config.py` 是整个系统的大脑。理解这里面的参数，你就掌握了量化策略的核心逻辑。

```python
# Tushare 接口密钥，用于获取合法的数据拉取权限


# 回测与数据获取的起止时间 (格式: YYYYMMDD)
START_DATE = '20180101'
END_DATE = '20231231'

# 股票池设定：当前设置为 000300 (沪深300指数的成分股)
INDEX_CODE = '000300'

# ==========================================
# 核心一：因子参数 (特征工程)
# ==========================================
# LOOKBACK_WINDOWS = [5, 10, 20, 60] 
# 这四个数字代表我们在计算技术指标时“往回看多少天”。
# 5代表一周(短线)，10代表两周，20代表一个月(中线)，60代表一个季度(长线)。
# 系统会自动为每个窗口计算动量、波动率、均线偏离度等。例如：
# - mom_5: 过去5天的收益率 (动量)
# - volatility_20: 过去20天收益率的标准差 (波动率)
# - bias_60: 当前价格偏离60日均线的程度
LOOKBACK_WINDOWS = [5, 10, 20, 60]

# ==========================================
# 核心二：XGBoost 模型超参数
# ==========================================
XGB_PARAMS = {
    # 'reg:squarederror': 我们使用的是回归模型(预测具体的收益率数值，而不是分类涨跌)
    'objective': 'reg:squarederror',
    
    # 学习率：模型每次学习时更新的步长，0.05 属于偏保守、稳健的数值
    'learning_rate': 0.05,
    
    # 树的最大深度：控制模型的复杂度。5 表示树不会太深，防止对训练数据“死记硬背”(过拟合)
    'max_depth': 5,
    
    # 树的数量：总共构建 100 棵决策树来进行集成预测
    'n_estimators': 100,
    
    # 行采样率：每棵树只随机抽取 80% 的历史数据进行训练，增加模型的泛化能力
    'subsample': 0.8,
    
    # 列采样率：每棵树只随机抽取 80% 的特征(因子)进行训练
    'colsample_bytree': 0.8,
    
    # 随机种子：保证每次运行代码，模型训练的结果是固定可复现的
    'random_state': 42
}

# ==========================================
# 核心三：回测系统参数 (交易规则)
# ==========================================
# 初始模拟资金：1,000,000 元 (一百万)
INITIAL_CAPITAL = 1000000.0

# 交易手续费(双边)：0.001 代表千分之一。
# 实际A股包含：印花税(卖出千分之0.5)、券商佣金(约万分之二)、过户费等，综合千一是一个合理的估算。
COMMISSION_RATE = 0.001 

# 调仓周期：30 代表每隔 30 个交易日（约一个半月），系统会强制卖出旧股票，买入新股票。
REBALANCE_PERIOD = 30 

# 每次调仓持有的股票数量：10。
# 模型会给所有股票的未来预期收益打分，系统只买排名前 10 的股票，每只股票分配 10% 的资金(等权重)。
TOP_K = 10 
```

## 3. 因子与标签是什么？

在量化机器学习中，我们需要告诉模型两件事：**“根据什么猜” (特征/因子)** 和 **“猜什么” (标签/目标)**。

### 因子 (Features)
在 `factor_calculator.py` 中，系统基于基础量价数据计算了以下几类因子（配合上面的 `LOOKBACK_WINDOWS` 会生成几十个特征列）：
*   **动量因子 (Momentum)**：过去 N 天涨得好不好？（追涨杀跌逻辑）
*   **波动率 (Volatility)**：过去 N 天的涨跌幅度剧烈吗？（风险度量）
*   **乖离率 (Bias)**：现在价格偏离 N 日均线有多远？（均值回归逻辑）
*   **成交量比 (Volume Ratio)**：近期成交量相比长期平均是否放大？（资金活跃度）
*   **MACD**：经典的趋势跟踪指标。

### 标签 (Target Label)
模型的目标是预测未来。在 `factor_calculator.py` 的这行代码中定义了模型的“考题”：
```python
target_window = 30
df['target_return'] = df['close'].shift(-target_window) / df['close'] - 1.0
```
这意味着：模型在学习时，会看到某一天的几十个因子，同时也会被告知“这只股票在 **未来30天后** 真实的涨跌幅（`target_return`）是多少”。模型通过海量历史数据的学习，试图找出这些因子与未来30天涨幅之间的数学关系。

## 4. 如何运行与测试

### 运行环境准备
确保你已经安装了 Python 3.9+，并在项目根目录下安装了所有依赖：
```bash
pip install -r requirements.txt
```

### 启动系统
在终端中执行：
```bash
python main.py
```
**运行流程**：
1. **数据拉取**：系统会自动获取沪深300成分股列表，并逐一拉取日线数据。第一次运行较慢（需几分钟），完成后会在本地生成 `raw_data.csv`。第二次运行将直接读取本地缓存，秒速跳过此步。
2. **计算因子**：处理原始数据，生成特征和标签，保存为 `factor_data.csv`。
3. **模型训练**：将前 80% 的时间作为训练集，后 20% 作为测试集。训练 XGBoost 模型并对测试集进行预测，结果保存至 `predictions.csv`。
4. **执行回测**：读取预测结果，按照 `config.py` 中的规则进行 30 日周期的模拟交易。
5. **输出报告**：在终端打印策略的最终收益率与基准收益率，并在项目目录下生成图表 `backtest_result.png`。

## 5. 如何自行扩展与进阶优化？

目前的系统是一个非常标准的量化框架“骨架”。如果你想让它在实盘中真正赚钱，可以尝试以下方向的改造：

### 📈 进阶一：引入基本面因子 (财务数据)
当前系统只有“量价因子”（技术分析）。你可以通过 Tushare 获取上市公司的财报数据，将其作为新的特征加入到 `factor_calculator.py` 中。
*   **估值因子**：PE（市盈率）、PB（市净率）、PS（市销率）
*   **盈利因子**：ROE（净资产收益率）、净利润同比增长率
*   *思路*：将量价因子的“短期博弈”与基本面因子的“长期价值”结合，模型选出的股票会更靠谱。

### 🛡️ 进阶二：完善回测机制 (加入风控)
目前的 `backtester.py` 是“死扛”模式（买入后无论如何都持有30天）。
*   **加入止损**：在回测循环中，每天检查持仓的 `next_day_return` 累计值，如果某只股票亏损超过 8%，立刻在次日开盘卖出。
*   **加入大盘择时**：当检测到沪深300指数的 20 日均线向下时，系统强制空仓（不选股），规避系统性熊市风险。

### 🤖 进阶三：优化机器学习模型
*   **调参 (Hyperparameter Tuning)**：使用 `GridSearchCV` 或 `Optuna` 自动寻找 `XGB_PARAMS` 中最完美的参数组合。
*   **换用分类模型**：目前是预测具体的收益率数值（回归）。你可以把标签改为“未来是否跑赢大盘”（1代表跑赢，0代表跑输），将模型改为分类器（`XGBClassifier`），有时候分类模型的胜率更高。
*   **防止未来函数**：仔细检查 `factor_calculator.py`，确保今天计算出的所有因子，都绝对没有用到“明天”及以后的数据（例如避免用到复权后导致失真的旧价格）。

### 🚀 进阶四：扩大选股池
在 `config.py` 中将 `INDEX_CODE` 改为中证500（`000500`）或中证1000（`000852`）。通常来说，机器学习在中小盘股（量化私募的主战场）中挖掘超额收益（Alpha）的能力，远强于在成熟透明的沪深300大盘股中。# AI-agent
